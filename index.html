Imagine a "typical" learning experience...

Typical training looks like...

  • Sage on the stage
  • Passive consumption, or follow along
  • Focus is on remembering and repeating information
  • Assessments don't resemble the skill
  • Synchronous, inflexible
  • Linear syllabus
  • Too fast or too slow
  • Learners have little agency

Who decided it should be like that?

Because

We know more now

Because

We have better tools

Because

We have different aims

Designing learning experiences from first principles

A normal classroom

Bloom's 2 sigma problem

The average student tutored one-to-one using mastery learning techniques performed two standard deviations better than students educated in a classroom environment

Why is this a "problem"?

Mechanisms

  • One on one tutoring
  • Mastery learning techniques

Mastery based learning

How we do it

Asynchronous learning

  • If you need to take more time - take it
  • If you need help - get it as you need it
  • Fill the gaps
  • If you can move forward quickly - more power to you
  • Extra challenges
  • No need for everyone to learn the same thing
  • (Except for teamwork training)

Efficiency supported by

  • Damn fine content
  • Skilled educators
  • Quick response times

The learning resembles the work

  • Super practical and hands on
  • Goodhart's law: When a measure becomes a target, it ceases to be a good measure

Mastery cannot be bought

  • Zone of proximal development
  • Focus on full understanding
  • Focus on empowering the learners - for continued growth

Trust the learner

  • Learners manage their own attention
  • We are all adults here - with agency, goals, motivation...

Challenges

  • Motivation
  • Connection

Motivation

  • Frontload personal relevance
  • Connection helps
  • Aim to get people to keep learning after the experience is over

Connection

  • Async chat - personal, project, content
  • Expert help on tap
  • Synchronous sessions (video calls)
  • Learners can talk to each other about the work
  • Learners can nerd out about all kinds of things
  • Space for co-working / body-doubling

Other Best practices

  • Spaced repitition
  • Recall / Apply
  • Ask then tell
  • Structure building

Practically speaking

  • Chat app: Talk to your peers and instructor whenever you need to
  • Quick turnaround time when you need help
  • Video calls, screen shares, pair programming, and text chat available
  • Frequent optional facilitated group calls
  • Get all the (damn fine) content up front
  • Different people can be learning different things
